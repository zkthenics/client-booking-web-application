# 🎬 CineBook — Cinema Booking Web App

A small web app for browsing films and booking cinema seats. Built with the exact
stack from our training: **Node.js + Express + EJS + Bootstrap 5.3** (plus a little
browser JavaScript). No database — bookings are carried through the URL and hidden
form fields, which keeps it simple to run and maintain.

**The user journey (4 screens + 2 popups):**

```
Home (film grid)  ──click a film──►  Film popup (details + showtimes)
        │                                      │
        │                              click a showtime
        ▼                                      ▼
   (R1 Movie listing)                  Seat map page  ──"Continue"──►  Checkout popup
                                       (R3 Seat booking)               (R4 Checkout)
                                                                              │
                                                                       "Confirm"
                                                                              ▼
                                                                   Ticket / Booking summary
                                                                   (R5 — reference CIN-####)
```

---

## ▶️ Run it (everyone, after cloning)

```bash
npm install        # installs Express, EJS, nodemon (one time)
npm start          # starts the server with auto-reload
```

Then open **http://localhost:3000** in your browser. Stop the server with `Ctrl + C`.
That's it — no setup, no accounts, no database.

> If port 3000 is busy, change `PORT` near the bottom of `app.js` to `3001`.

### The 3 "shared" files (don't fight over these — talk first)
- **`dataService.js`** — the data layer. Every page asks it for films/showtimes/seats.
  It already has all the functions you need (see comments inside). You rarely change it.
- **`app.js`** — server setup + two helpers every page can use:
  `ratingClass()` (colours the age badges) and `relativeDay()` (turns a date into
  "Today" / "Tomorrow" / "Thu 26 Jun").
- **`styles.css`** — one shared stylesheet. The colours/fonts live in the `:root`
  tokens at the very top; change a token there and the whole site updates.

### ⚠ The one thing that trips everyone up — the data model
The JSON names are misleading, so remember:
- `movies.json` → the films. ✅ as expected.
- `showings.json` → **seat layouts / rooms** (rows of seats). ❌ NOT show dates.
- `times.json` → **the actual schedule**. Each row joins a `movieId` + `showingId`
  (which room) + a `date` + a `time`. This is the file you filter to get a film's showtimes.

You never touch the JSON directly — always go through `dataService.js`.

## 🌱 Branches — keep it simple (one branch each, named after you)

No `feature/…` branches. Just **your first name**: `zak`, `abid`, `nico`, `shabir`, `zahraa`.
You do all your work on your own branch and merge it into `main` when a piece works.

### One-time team setup (ONE person does this — probably Zak)
Create the empty repo on GitHub first (no README), then from inside this folder:

```bash
git init
git add .
git commit -m "Initial CineBook setup"
git branch -M main
git remote add origin git@github.com:<team-username>/<repo-name>.git
git push -u origin main
```

Then on GitHub: **Settings → Collaborators → add the other four**.

### Everyone else (first time)
```bash
git clone git@github.com:<team-username>/<repo-name>.git
cd <repo-name>
npm install
git checkout -b <yourname>      # e.g. git checkout -b nico
```

## 🔁 The everyday workflow — commit → push → merge to main

Every change you make follows the same loop. Full details (with pictures of what each
command does) are in **`docs/git-workflow.md`** — but the short version is:

```bash
# 1. Start the day on the latest shared code
git checkout main
git pull
git checkout <yourname>         # back to your branch
git merge main                  # pull everyone's merged work into yours

# 2. ...do your work, then save it...
git add .
git commit -m "R3: seats turn red when selected"
git push                        # first push: git push -u origin <yourname>

# 3. When a piece is finished → merge it into main
#    Open a Pull Request on GitHub, set Zahraa as reviewer, then Merge.
```

**Golden rules:** pull before you start, commit small & often, never push broken code
to `main`, and if two people must share a file, message each other first.

## 🛠 Troubleshooting (quick)

| Problem | Fix |
|---|---|
| `command not found: npm` | Install Node (we used **nvm** + Node v26 in training). |
| Port 3000 already in use | Change `PORT` in `app.js` to `3001`, or stop the other server. |
| Page won't load / 500 error | Look at the terminal — the error + file + line number is printed there. |
| Popups won't open | The Bootstrap JS in `footer.ejs` must be present — don't delete it. |
| Merge conflict (`<<<<<<<`) | See the "merge conflicts" section in `docs/git-workflow.md`. |
